有康文件拆分服务 - 部署手册

提示：本文档面向运维人员，请按顺序执行每一步。

一、准备工作

1.1 确认服务器已安装 Java 17

在 Linux 服务器上执行：

java -version

如果显示类似下面的内容，说明已安装：

openjdk version "17.0.x" 20xx-xx-xx

如果未安装，请先安装 Java 17（联系管理员或使用包管理器安装）。

1.2 确认 jar 包已准备好

从开发人员处获取 youkang-file-splitter.jar 文件。

二、创建必要的文件夹

在 Linux 服务器上依次执行以下命令，直接复制粘贴即可：

# 创建应用安装目录
mkdir -p /opt/youkang-file-splitter

# 创建数据目录（程序运行时需要的文件夹）
mkdir -p /data/youkang/inbox
mkdir -p /data/youkang/work
mkdir -p /data/youkang/output
mkdir -p /data/youkang/failed
mkdir -p /data/youkang/archive

# 创建日志目录
mkdir -p /data/youkang/splitter/logs

文件夹说明：

文件夹路径 用途
/opt/youkang-file-splitter 存放 jar 包
/data/youkang/inbox 用户上传 ZIP 文件的入口目录
/data/youkang/work 程序解压和处理时的临时工作目录
/data/youkang/output 拆分完成后的文件输出目录
/data/youkang/failed 处理失败的文件归档目录
/data/youkang/archive 处理成功的原始文件归档目录
/data/youkang/splitter/logs 程序运行日志目录

三、上传 jar 包到服务器

以下提供 3 种方式，选择任意一种即可。

方式一：使用 scp 命令（推荐）

在本地电脑（Windows 或 Mac）的终端/PowerShell 中执行：

scp youkang-file-splitter.jar root@服务器IP:/opt/youkang-file-splitter/

执行后会提示输入服务器密码，输入即可。

示例（假设服务器 IP 是 192.168.1.100）：

scp youkang-file-splitter.jar root@192.168.1.100:/opt/youkang-file-splitter/

方式二：使用 FTP/SFTP 工具（图形界面）

使用 FileZilla、WinSCP 等工具连接服务器，将 youkang-file-splitter.jar 上传到 /opt/youkang-file-splitter/ 目录。

方式三：使用 rz 命令

在 Linux 服务器上执行：

cd /opt/youkang-file-splitter
rz

然后会弹出文件选择窗口，选择本地的 youkang-file-splitter.jar 文件即可上传。

四、创建 systemd 服务

systemd 服务可以实现开机自启和崩溃自动重启。

4.1 创建服务配置文件

在 Linux 服务器上执行以下完整命令（直接复制粘贴）：

cat > /etc/systemd/system/youkang-file-splitter.service << 'EOF'
[Unit]
Description=Youkang File Splitter Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/youkang-file-splitter
ExecStart=/usr/bin/java -jar youkang-file-splitter.jar --spring.profiles.active=prod
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

4.2 重新加载系统服务配置

systemctl daemon-reload

五、启动服务

5.1 启动程序

systemctl start youkang-file-splitter

5.2 查看是否启动成功

systemctl status youkang-file-splitter

如果显示 active (running)，说明启动成功：

● youkang-file-splitter.service - Youkang File Splitter Service
   Loaded: loaded (/etc/systemd/system/youkang-file-splitter.service; ...)
   Active: active (running) since Mon 2024-01-15 10:00:00 CST; 5s ago

如果显示 failed，请查看日志排查问题（见第六节）。

5.3 设置开机自动启动

systemctl enable youkang-file-splitter

这样服务器重启后，程序会自动启动，无需手动干预。

六、常用运维命令

操作 命令
查看运行状态 systemctl status youkang-file-splitter
停止程序 systemctl stop youkang-file-splitter
重启程序 systemctl restart youkang-file-splitter
查看实时日志 journalctl -u youkang-file-splitter -f
查看历史日志 journalctl -u youkang-file-splitter
查看主日志文件 tail -f /data/youkang/splitter/logs/splitter.log
查看任务日志 tail -f /data/youkang/splitter/logs/splitter.log.task

提示：按 Ctrl + C 退出日志实时查看。

七、部署验证

7.1 检查进程是否在运行

ps -ef | grep youkang-file-splitter

应该能看到 java 进程。

7.2 检查日志是否正常输出

tail -20 /data/youkang/splitter/logs/splitter.log

应该能看到程序启动的日志信息。

7.3 测试文件拆分功能

  1. 准备一个包含生信产物的文件夹
  2. 将文件夹放入 /data/youkang/inbox 目录
  3. 等待 1 分钟左右
  4. 检查 /data/youkang/output 目录是否有拆分后的文件
  5. 检查 /data/youkang/archive 目录是否有原始文件夹归档

八、更新程序（后续升级）

当开发人员提供新版本 jar 包时：

# 1. 停止旧版本
systemctl stop youkang-file-splitter

# 2. 上传新的 jar 包到 /opt/youkang-file-splitter/
# （参考第三节的上传方式）

# 3. 启动新版本
systemctl start youkang-file-splitter

# 4. 查看状态确认正常
systemctl status youkang-file-splitter

九、常见问题

Q1: 启动失败，提示 "Failed to start"

排查步骤：

# 查看详细错误信息
journalctl -u youkang-file-splitter -n 50

常见原因：

  • Java 未安装或版本不对（需要 Java 17）
  • jar 包没有上传成功或路径不对
  • /data/youkang 目录没有创建

Q2: 程序运行了但没有处理文件

检查：

# 检查 inbox 目录权限
ls -la /data/youkang/inbox

# 检查日志是否有报错
tail -50 /data/youkang/splitter/logs/splitter.log

Q3: 如何修改扫描频率或其他配置

配置文件在 jar 包内部，不能直接修改。需要联系开发人员重新打包，或通过在启动命令中添加参数覆盖。

十、目录结构总览

部署完成后，服务器上的目录结构如下：

/opt/
└── youkang-file-splitter/
    └── youkang-file-splitter.jar          # 程序文件

/data/
└── youkang/
    ├── inbox/                             # 上传入口
    ├── work/                              # 临时工作目录
    ├── output/                            # 拆分结果输出
    ├── failed/                            # 失败文件归档
    ├── archive/                           # 成功文件归档
    └── splitter/
        └── logs/
            ├── splitter.log               # 主日志
            └── splitter.log.task          # 任务日志

/etc/systemd/system/
└── youkang-file-splitter.service          # 系统服务配置

如有问题，请联系开发人员。
